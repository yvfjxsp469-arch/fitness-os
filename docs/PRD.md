# Fitness OS — Product Requirements Document (v2.0)

## 1. 产品概述

### 1.1 产品愿景

Fitness OS 是一个**个人健身操作系统**，将减脂过程中产生的全部数据（体重、围度、饮食、训练、目标）汇集到统一平台，通过数据驱动的方式帮助用户从 115kg 减脂到 90kg，并在达成目标后继续作为长期健康管理系统使用。

### 1.2 核心问题

| 痛点 | 描述 |
|------|------|
| 数据碎片化 | 体重记在体重秤 App、饮食记在薄荷、训练记在备忘录，数据无法关联分析 |
| 缺乏因果洞察 | 不知道体重波动是因为吃咸了、睡差了、还是训练强度不够 |
| 动力衰减 | 没有里程碑反馈，减脂中后期容易放弃 |
| 训练盲区 | 不知道卧推有没有进步、RDL 有没有进步、过去三个月训练容量变化 |
| 身体成分盲区 | 只关注体重，无法区分"减脂"和"掉秤"（腰围下降但体重不降=正在减脂） |
| 没有「个人看板」 | 无法一眼看清今天的完成度 |

### 1.3 目标用户

**唯一用户：开发者本人（115kg → 90kg）**

- 当前体重：115kg
- 目标体重：90kg
- 需减重：25kg
- 预计周期：6-8 个月（每周 0.7-1.0kg）
- 训练模式：Push / Pull / Legs 三分化
- Apple Watch 使用者（未来接入健康数据）
- 使用场景：每天多次（早晨称重、三餐记录、训练记录、睡前复盘）

### 1.4 成功指标

| 指标 | 目标值 | 衡量方式 |
|------|--------|----------|
| 体重下降 | 115kg → 90kg | 每日称重记录 |
| 数据录入率 | > 90% 天数完成核心数据录入 | 系统统计 |
| 热量合规率 | > 80% 天数在目标热量范围内 | 摄入 vs 目标对比 |
| 训练完成率 | > 80% 计划训练完成 | 训练日志 vs 计划 |
| 腰围下降 | < 90cm（最终） | 每周测量 |
| 力量保持 | 减脂期三大项不低于起始 80% | 训练 PR 记录 |

## 2. 减脂方法论（系统内置逻辑）

### 2.1 能量平衡模型

```
体重变化 = 摄入热量 - 消耗热量（TDEE）

TDEE = BMR × 活动系数

BMR (Mifflin-St Jeor):
  男性: 10 × 体重(kg) + 6.25 × 身高(cm) - 5 × 年龄 + 5
  女性: 10 × 体重(kg) + 6.25 × 身高(cm) - 5 × 年龄 - 161

活动系数:
  1.2   — 久坐（几乎不运动）
  1.375 — 轻度活动（每周 1-3 天）
  1.55  — 中度活动（每周 3-5 天）
  1.725 — 高度活动（每周 6-7 天）
  1.9   — 极高强度（体力劳动 + 训练）
```

### 2.2 宏观营养素分配

| 营养素 | 占比 | 每克热量 | 说明 |
|--------|------|----------|------|
| 蛋白质 | 35-40% | 4 kcal | 每 kg 体重 1.6-2.2g，防止肌肉流失 |
| 碳水化合物 | 30-35% | 4 kcal | 训练日适当调高 |
| 脂肪 | 25-30% | 9 kcal | 每 kg 体重 0.6-1.0g，维持激素水平 |

### 2.3 减脂节奏

| 阶段 | 体重范围 | 周期 | 每周减重 | 热量缺口 | 里程碑 |
|------|----------|------|----------|----------|--------|
| 快速启动期 | 115-105kg | 8-10 周 | 1.0kg | 1000 kcal | 110kg ✓ |
| 稳定减脂期 | 105-95kg | 10-12 周 | 0.8kg | 800 kcal | 100kg ✓ |
| 冲刺塑形期 | 95-90kg | 8-10 周 | 0.5kg | 500 kcal | 95kg → 90kg ✓ |

## 3. MVP 定义

### 3.1 MVP 原则

> **一个人、一条减脂主线、训练数据不可妥协。**

MVP 核心：**能量记账 + 动作级训练记录**。记录体重、饮食热量、训练动作数据，系统提供进度洞察和目标追踪。

### 3.2 MVP 范围（v1.0）

#### 包含（In Scope）

| 模块 | 功能 | 优先级 |
|------|------|--------|
| 🔐 认证 | 单用户注册/登录（bcrypt + JWT cookie） | P0 |
| 🏠 仪表盘 | 今日体重+趋势 mini 图、热量进度条+宏观营养素环、本周训练次数、连续打卡天数、当前减脂阶段、最近里程碑 | P0 |
| ⚖️ 体重 | 每日称重 CRUD、趋势图（7/30/90天+7日移动平均+目标参考线）、滚动7日均值、统计卡片 | P0 |
| 🍽️ 饮食 | 每餐记录热量+三大宏量+纤维、食物库（预设 8 分类 + 自定义）、今日摄入 vs 目标 | P0 |
| 🏋️ 训练 | **动作级记录**：训练 → 动作 → 组数×重量×次数、训练模板（Push A / Pull A / Legs A 预设）、PR 自动追踪、日历视图 | P0 |
| 🎯 目标 | 目标体重 115→90 + **自动生成里程碑**（110/105/100/95/90）、进度条、预计达成日期 | P0 |
| 📏 围度 | 腰围 + 胸围记录、围度趋势图、周报含围度变化 | P1 |
| 📊 周报 | 每周均重、热量日均、训练次数+容量、体重变化、腰围变化、下周建议 | P1 |

#### 不包含（Out of Scope for MVP）

- AI 饮食建议 / 食谱生成
- Apple Watch 数据接入（仅预留数据结构）
- 条形码扫描
- 体脂率测量（设备不支持）
- 睡眠追踪
- 习惯打卡系统
- 进度照片
- 数据导出
- PWA / 离线模式

### 3.3 MVP 页面清单

```
/                  → 仪表盘（今日概览）
/weight            → 体重记录 + 趋势图
/nutrition         → 饮食记录
/nutrition/foods   → 食物库管理
/workout           → 训练记录（动作级）
/workout/templates → 训练模板管理
/workout/exercises → 动作库管理
/measurements      → 身体围度记录
/goals             → 目标 + 里程碑
/weekly-report     → 周报
/login             → 登录/注册
```

### 3.4 MVP 数据流

```
早晨称重    → 录入体重           → 更新趋势图 → 重新计算 TDEE → 更新里程碑进度
三餐时间    → 选择食物库录入饮食  → 更新热量+宏观进度条 → 显示剩余
训练前      → 选择模板开始训练    → 自动生成 Workout 记录
训练中      → 逐组记录重量×次数   → 检测是否 PR
训练后      → 标记完成            → 更新本周统计 → 更新 PR 记录
每周日晚    → 自动生成周报        → 评估减脂速率 → 调整下周热量目标
```

### 4.2 训练模板设计（v1.0 内置）

用户训练为 Push / Pull / Legs 三分化，系统首次启动时自动创建：

**Push A**（胸+肩+三头）
1. 卧推
2. 上斜哑铃卧推
3. 飞鸟
4. 器械肩推
5. 绳索侧平举
6. 绳索臂屈伸

**Pull A**（背+二头+后束）
1. 对握高位下拉
2. 胸托划船
3. 单臂悍马划船
4. 宽握坐姿划船
5. 面拉
6. 上斜哑铃弯举

**Legs A**（腿+核心）
1. 前蹲
2. 保加利亚分腿蹲
3. 罗马尼亚硬拉
4. 腿弯举
5. 提踵

训练时：点击模板 → 自动创建 Workout + WorkoutExercise → 逐组填写重量×次数。

## 6. 技术约束

| 约束 | 选择 |
|------|------|
| 用户系统 | 单用户，保留扩展能力 |
| 数据归属 | 用户拥有全部数据，Supabase PostgreSQL |
| 响应式 | 移动优先（训练和饮食记录场景多在手机端） |
| 国内网络 | 避免 Google Fonts、YouTube 等被墙资源；Recharts 自托管 |
| 离线 | MVP 不做离线 |
| AI API | v2.5 接入 DeepSeek API |
| HealthKit | v1.3 预留 HealthMetric 表结构 |
